01 / O Centro Espírita

O que é o N.E.E.E. Emmanuel?

Fundado em 1969 em Mairiporã - SP, o Núcleo de Estudos Espíritas Emmanuel é uma instituição filantrópica que promove o estudo e a prática da doutrina espírita, além de prestar relevante apoio social à comunidade local.

  • Estudo Doutrinário: Cursos sistemáticos e palestras públicas regulares.
  • Bazar Solidário: Captação de recursos via brechó interno ("Breshopping").
  • Biblioteca: Difusão de livros espíritas clássicos e contemporâneos.
  • Conversa Fraterna: Atendimento espiritual e acolhimento emocional.

02 / O Desafio

Modernização Sem Servidor

O desafio consistiu em desenhar e implementar um sistema interativo para gerir os serviços de assistência, empréstimos e vendas com restrições severas de infraestrutura.

  • Custo Zero de Hospedagem: Operação inteiramente estática (Client-Side).
  • Segurança por Isolamento: Informações críticas dos leitores e agendamentos mantidas apenas localmente.
  • Simulação de Alta Fidelidade: Operação reativa contendo controles e validações realistas de um banco de dados dinâmico.

05 / Implementação

Lógica de Checkout

Redução dinâmica do estoque após validação da sacola. O cálculo consolida descontos de forma reativa no ato do encerramento.

  • Atualiza o array de estoque global em memória.
  • Aplica desconto e formata o comprovante.
  • Invoca a atualização visual e persiste os dados.
portal.js > checkout() 
function checkout() {
    if (cart.length === 0) return;
    cart.forEach(item => {
        const idx = products.findIndex(p => p.id === item.id);
        if (idx !== -1) {
            products[idx].stock -= item.qty;
        }
    });
    const subtotal = cart.reduce((a,i)=>a+(i.price*i.qty), 0);
    const disc = applyDiscountCb.checked ? subtotal * 0.1 : 0;
    showReceipt(subtotal, disc, subtotal - disc);
    cart = [];
    saveState();
}

07 / Implementação

Fluxo de Reserva

O envio do formulário altera reativamente o status do livro físico e registra a intenção de retirada, atualizando a fila do dashboard do leitor.

  • Previne o recarregamento com `e.preventDefault()`.
  • Gera um ID único usando timestamp (`Date.now()`).
  • Realiza renderização subsequente de atualização.
portal.js > submitReserve 
formBookReserve.addEventListener("submit", function(e) {
    e.preventDefault();
    const bookId = parseInt(document.getElementById("modal-book-id").value);
    const name = document.getElementById("reserve-name").value;
    const idx = books.findIndex(b => b.id === bookId);
    
    if (idx !== -1 && books[idx].status === "Disponível") {
        books[idx].status = "Reservado";
        reservas.push({ id: Date.now(), bookId, readerName: name });
        saveState();
        renderBooks();
        closeReserveModal();
    }
});

08 / Agendamentos

Acolhimento Fraterno

O Acolhimento Fraterno é uma conversa individual e privativa de amparo. O portal simula a solicitação respeitando as regras físicas de calendário.

  • Regra do Dia Fixo: Os atendimentos no centro físico ocorrem estritamente às terças-feiras. O formulário rejeita outros dias.

  • Gestão de Escalas: Centralização de voluntários e atribuição rápida de tarefas de apoio.

09 / Implementação

Validação Temporal

O processamento do formulário de agendamento bloqueia datas inválidas por meio de operações sobre objetos `Date` nativos do JavaScript.

  • Compara o timestamp selecionado com o início do dia de hoje.
  • Usa `getDay() !== 2` para rejeitar datas fora de terças-feiras.
portal.js > dateValidation 
const parts = dateVal.split("-");
const selectedDate = new Date(
    parseInt(parts[0]), 
    parseInt(parts[1]) - 1, 
    parseInt(parts[2])
);
const today = new Date();
today.setHours(0,0,0,0);

if (selectedDate < today) {
    alert("Selecione uma data futura."); return;
}
if (selectedDate.getDay() !== 2) {
    alert("O acolhimento ocorre apenas às terças-feiras."); return;
}

10 / Persistência

Banco de Dados Local

Simulação robusta de persistência. A manipulação de arrays é sincronizada com o armazenamento persistente do navegador, evitando perda de estado.

  • Uso do `localStorage` da Web Storage API.
  • Serialização e desserialização rápida via objetos JSON.
  • Isolamento de escopo por meio de IIFE auto-executável.
portal.js > localStorage 
// Estado inicializado a partir do localStorage
let books = JSON.parse(
    localStorage.getItem("neee_books")
) || defaultBooks;

function saveState() {
    localStorage.setItem("neee_books", JSON.stringify(books));
    localStorage.setItem("neee_products", JSON.stringify(products));
    localStorage.setItem("neee_reservas", JSON.stringify(reservas));
    updateStats();
}

11 / Interfaces

Impressão & Responsividade

O Portal é 100% responsivo para celulares e computadores. Para o bazar de caridade, uma folha de estilo de impressão isola o cupom fiscal físico.

  • Injeta CSS temporário de impressão via DOM.
  • Esconde elementos da tela e exibe a guia com formatação de bobina térmica.
  • Dispara `window.print()` e remove a folha temporária.
portal.js > printReceipt() 
window.printReceipt = function() {
    const html = document.getElementById("receipt-print-area").innerHTML;
    const style = document.createElement("style");
    style.innerHTML = `
        @media print {
            body * { visibility: hidden; }
            #print-sect, #print-sect * { visibility: visible; }
            #print-sect { position: absolute; left:0; top:0; width:100%; }
        }
    `;
    document.head.appendChild(style);
    window.print();
};
